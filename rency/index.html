con·cur·ren·cy

In computer science, concurrency is a property of systems in which several computations are executing simultaneously, and potentially interacting with each other 
Synonyms
  coexistence, coincidence, concurrence
Antonyms
  conflict, disagreement, dissensus

You may not worry about concurrency if:

  • You use Rails
  • Your application is single-threaded
  • You have a very low traffic

Bonus: Interview Question

Design a ToDo application. Requirements:

  • User can add an item at an arbitrary place
  • User can easily reorder items on the list
  • All edits are saved "behind the scenes"

You get points for considering:

  • Synchronous vs. asynchronous messaging
  • Lost reordering messages
  • Reordering messages coming out-of-order
